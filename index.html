Queen's Space Conference | Microscopic Space Debris Case Study

Feb 2022

Queen’s Space Conference is a university run event that hosts prominent members of the space industry, and they speak about their experiences and research. The event also offers a case study event, where students work in randomly assigned teams to find a solution to a space related problem.

The case study involved developing a solution to the issue of tiny metal particles in the orbital field of earth that are not being disposed of. My team’s solution involved using an expanding foam solution combined with an electromagnet to pull tiny metal particles into the foam where they would consolidate. Once enough material had been caught, the device would naturally fall into the Earth's atmosphere and burn up, as when an orbiting device gains mass, it's orbital radius gets smaller.

Skills:

  • Presenting to judges
  • Orbital physics
  • Design
  • Teamwork

Achievements:

  • 1st place in the competition
  • Well received by members of the space industry

Vertical Farm

Sep - Dec | 2021

As part of engineering coursework, students were tasked with designing a multi floor vertical farm including prototyping an elevator.

My part of the project involved designing a pulley system and creating a code that controlled the elevator. I used a double pulley design with a counterweight to improve the efficiency of the elevator. The code was made using C++ using an Arduino to control the motor and sensors. The design used math to convert the rotary distance moved by the gears into linear distance moved by the elevator. Then the height of each floor was provided, and the number of rotations required by the motor was calculated and applied based on how many floors the elevator needed to travel.

Skills:

  • Classical physics
  • Sensor control
  • C++
  • Mechanical design

Autonomous Rover LiDAR | Point Cloud

Jan - April | 2022

Students were required to get engineering experience by solving an assigned problem for a stakeholder. My team and I were tasked with designing and building code in python to autonommously control a rover, which would be used by the Queen's Space Engineering Team (QSET).

My part of the project involved taking LiDAR data (laser sensor that detects the distance and angle to objects) and processing it into a point cloud. Point clouds are a way of storing position information on a plane, identifying each individual point and finding the distance between each point and the next closest point. This creates an understanding of where the robot can move to, and which objects it needs to avoid.

Skills:

  • Python
  • Point cloud
  • LiDAR
  • Autonomous coding

Outcome:

Project ultimately not successful. My point cloud system code worked properly, but other portions of the code didn't work as expected. The algorithm could only turn right, drastically reducing efficiency, and would often misjudge its width and collide with objects.

Goniophotometer and Characterization

Jan - April | 2024

For an optics course, students were tasked with building a simple goniophotometer, and characterizing a photodiode.

My part of the project focused on the design and programming of the goniophotometer. The main components involved a step motor control, a 3d printed base, and a rotating piece. The design had a light source that was held in place using the base, and the rotating piece would rotate around the LED to get a profile of where the light was strongest. Note that real world goniophotometers rotate in two axes, whereas we only rotated in one axis for the scope of the project.

To control the motor I used a method called wave drive. Step motors have phases, where each phase you want to activate one after the other, which forms a wave pattern. After creating a wave pattern, I recorded the time it took for a full rotation, and wrote a code so that the motor would record its angle, so that it could move to whichever angle specified. Then I wrote a command that rotated it from left to right while recording the voltage readings from the photodiode.

I also worked with another teammate on building the characterization circuits. These were very complicated, as there were many components and small tweaks that needed to be made. The circuit used a low pass filter to slowly increase voltage; the circuitry causes the duty cycle (ratio of high voltage to low voltage from an AC source) to steadily increase. This results in a voltage vs time graph that resembles a right angle triangle.

Skills:

  • Photonics
  • Optics
  • CAD Design
  • Motor Control
  • C++
  • LabVIEW
  • Circuit Design

Outcome:

This project was mostly successful, with the mechanics and wiring all working properly, but the LabVIEW code for recording the data had some issues. Luckily we were still able to collect data, and the characterization part of the project was mostly successful as well.
